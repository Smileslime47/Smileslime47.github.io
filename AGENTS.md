# AGENTS.md

本文件是本仓库对 Codex / AI 代码代理的统一规则入口。

要求：

- 进入本仓库后，开始任何搜索、修改、重构、生成文件、提交之前，优先阅读本文件
- 本文件优先级高于散落在历史目录中的旧规则文件
- 若本文件与 README 或局部注释冲突，先暂停并向用户确认

## 1. 代理必须遵守的规则

### 1.1 编码与文件写回

- 默认只用 `apply_patch` 修改源码文件
- 如果 `apply_patch` 失败，不要直接退回 `Set-Content` / `Out-File`
- 在 Windows PowerShell 下，禁止对任何包含中文或可能包含非 ASCII 的源码文件使用 `Set-Content` / `Out-File` 直接回写
- 如果确实必须用 PowerShell 写文件，只能使用 `[System.IO.File]::WriteAllText(path, content, [System.Text.UTF8Encoding]::new($false))`
- 任何非 `apply_patch` 的源码写回后，必须立刻重新读取文件校验：
  - 中文未乱码
  - 没有 `�`
  - 没有异常 BOM
  - 副作用 diff 符合预期
- 未完成上述校验前，不得继续下一步
- 如果修复编码问题需要整体重写文件，先说明原因，再用显式 `UTF-8 无 BOM` 写回，并再次重读校验

### 1.2 修改策略

- 优先做小范围、可验证、可回退的修改
- 没有明确需求时，不要顺手改 unrelated 代码
- 如果发现工作区已有未提交改动，先识别是否与当前任务相关；不能擅自覆盖用户已有修改
- 非必要不要整体重写大文件
- 能抽局部工具函数时，不要上来就重构整片架构

### 1.3 沟通与提交

- 注释一律使用中文
- commit 信息一律使用中文
- 提交前确认工作区范围干净，避免把无关变更混进同一个提交
- 需要重写历史时，必须先明确说明影响

## 2. 项目整体规范

### 2.1 技术栈

- 使用 `TypeScript`、`Vue 3`、`Less`
- 路径别名 `@` 指向 `src`

### 2.2 Vue 约定

- `script` 默认使用 `<script setup lang="ts">`
- `style` 默认使用 `<style scoped lang="less">`
- 只有在确实需要覆盖全局样式时，才去掉 `scoped`

### 2.3 样式约定

- 优先复用 `src/styles` 下已有 less 变量、主题和公共样式
- 如果某段样式未来可能复用，优先抽到公共样式层，而不是直接散落在页面里
- 只有确认没有复用价值时，才写进组件自己的局部样式

### 2.4 注释与文案

- 代码注释使用中文
- 界面文案优先保持中文语境一致
- 不要引入乱码文本；如果发现已有乱码，处理时先确认影响范围

## 3. 项目代码组织约定

### 3.1 页面与布局

- 普通内容页优先复用 `src/component/common/ContentPageLayout.vue`
- 首页 `src/pages/HomePage.vue` 是特殊页面，不要轻易套用普通内容页模板
- 新增普通页面时，通常只需要：
  - 在 `src/pages/` 下创建页面
  - 在 `src/router/router.ts` 注册路由
  - 如有需要，再更新 `src/component/layout/Header.vue` 的导航入口

### 3.2 文章系统

文章系统核心位于 `src/service/posts/`：

- `loaders.ts`：文章和文章资源的按需加载入口
- `index-builder.ts`：文章摘要和分类树构建
- `frontmatter-parser.ts`：frontmatter 解析
- `repository.ts`：统一文章查询与缓存
- `types.ts`：文章相关类型定义

新增文章相关能力时，优先复用和扩展这层，不要在页面里重复写一套解析逻辑。

### 3.3 frontmatter 约定

当前主要使用这些字段：

- `title`
- `date`
- `publishedAt`
- `publishDate`
- `createdAt`
- `tags`

说明：

- 分类主要由 `src/posts` 下的目录结构承担
- `tags` 用于标签聚合
- 日期字段用于归档和排序

### 3.4 组件自动注册

项目启用了 `unplugin-vue-components`，会扫描：

- `src/component`
- `src/pages`

因此很多组件可以直接在模板中使用，不需要手动 `import`。

阅读和修改代码时，要先判断组件是否属于自动注册，不要误删看似“未使用”的组件引用方式。

## 4. 具体执行偏好

### 4.1 优先抽象复用的场景

出现以下情况时，优先考虑抽公共函数或工具模块：

- 标签归一化、日期格式化、摘要提取这类纯函数逻辑重复出现
- 主题读写、滚动调度、空闲执行这类跨页面逻辑重复出现
- 多个页面共享同一种 hero/meta/tags 展示结构

### 4.2 优先精简冗余的场景

出现以下情况时，优先考虑清理：

- 页面内为了展示而重复查找同一份数据
- 页面中存在已经不用的计算属性、样式、注释、字段
- 服务层已经提供能力，但页面仍然自行重复解析

## 5. 禁止事项

- 不要再把编码规则分散写到多个低可见位置，再指望代理自己猜
- 不要在未校验编码结果时继续后续修改
- 不要使用会引入 BOM 风险的 PowerShell 直写方式处理中文源码
- 不要提交英文 commit 信息
